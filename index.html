Abstract

Near-Earth Objects (NEOs) are solar system bodies whose orbital paths bring them into close proximity with Earth's orbit. Among these, Potentially Hazardous Asteroids (PHAs) — defined by estimated diameters exceeding 140 metres and Minimum Orbit Intersection Distances (MOID) of 0.05 AU or less — pose a scientifically significant risk to planetary safety. The accurate and timely automated classification of newly discovered NEOs is a critical component of global planetary defence strategy.

This paper presents a comprehensive machine learning-based binary classification system for NASA asteroid hazard prediction. Processing NASA's Nearest Earth Objects dataset of 4,687 cleaned records across 16 orbital and physical features, five supervised classifiers were trained and evaluated: Logistic Regression, Decision Tree, Random Forest, Gradient Boosting, and XGBoost. XGBoost achieved the highest performance with a ROC-AUC of 0.9992, Accuracy of 99.57%, and Hazardous-class Recall of 99.12%, validated by 5-fold stratified cross-validation (mean ROC-AUC: 0.9969 ± 0.0004). The trained model is deployed via a Gradio web application supporting real-time NASA Neo Reference ID lookup.

§ 1 Introduction

Near-Earth Objects (NEOs) are asteroids and comets whose orbital paths bring them within close proximity to Earth's orbit around the Sun — specifically within 1.3 Astronomical Units (AU). Of the millions of asteroids estimated to exist in the solar system, approximately 28,000 have been formally classified as NEOs by NASA's Center for Near Earth Object Studies (CNEOS).

A critically important subset — Potentially Hazardous Asteroids (PHAs) — are defined by two criteria: an estimated diameter exceeding 140 metres and a Minimum Orbit Intersection Distance (MOID) of 0.05 AU or less. The discovery rate of new NEOs is accelerating sharply; the Vera Rubin Observatory is projected to discover an estimated 5 million new NEOs within its first decade of operation — a volume that makes manual expert review practically infeasible.

Machine learning classifiers trained on the accumulated historical NEO catalogue offer a scalable, automated, and probabilistically rich alternative to threshold-based rule classification. Accurate hazard assessment directly informs telescope follow-up prioritisation, deflection mission planning, civil protection advisories, and international coordination through the IAWN and SMPAG.

"A volume of 5 million new NEO discoveries makes manual expert review practically infeasible — scalable ML classification becomes essential."

1.1 Objectives

The project aimed to: load and clean NASA's NEO dataset by removing 8 non-predictive administrative columns and 12 redundant correlated columns; generate exploratory visualisations including class distribution, diameter distribution, and full feature correlation heatmaps; train and compare five supervised binary classifiers; validate the best model via 5-fold stratified cross-validation and XGBoost feature importance analysis; and deploy a Gradio prediction interface supporting Neo Reference ID lookup.

§ 2 Existing System & Limitations

NASA's current methodology designates asteroids as Potentially Hazardous based on two fixed orbital mechanics thresholds: diameter > 140 m AND MOID ≤ 0.05 AU. While well-established, this approach has significant scalability and adaptability limitations.

2.1 Drawbacks of Threshold-Based Classification

No confidence estimation: Fixed thresholds produce a binary flag without associated probability. Objects near the boundary — diameters of 135–145 m or MOIDs of 0.048–0.052 AU — may be systematically misclassified when early-orbit estimates carry large uncertainties from limited observational arcs.

Scalability bottleneck: With the Vera Rubin Observatory expected to discover 5 million new NEOs in its first decade, manual expert review is infeasible at required throughputs.

No multi-feature interaction modelling: The existing approach evaluates diameter and MOID independently. Machine learning simultaneously captures complex non-linear interactions across all sixteen orbital and physical parameters — eccentricity, relative velocity, Jupiter Tisserand invariant, semi-major axis — encoding richer hazard signal.

No learning from historical data: A threshold system does not improve with accumulated observations. An ML pipeline can be periodically retrained on updated NEO catalogues, continuously improving discriminative accuracy near the hazard boundary.

3.1 Preprocessing Pipeline

Eight non-predictive administrative columns were removed (Neo Reference ID, Name, Orbit ID, Close Approach Date, Epoch Date, Orbit Determination Date, Orbiting Body, Equinox). Null value verification confirmed zero missing values. Twelve redundant correlated columns were dropped — duplicate diameter unit conversions, duplicate velocity measurements, and duplicate miss-distance units — retaining the most representative feature per group. The binary target 'Hazardous' was integer-encoded (True → 1, False → 0). A StandardScaler was fitted on training data only and applied to both splits.

3.2 Class Imbalance Handling

The 4.6:1 Non-Hazardous to Hazardous imbalance was addressed via stratified train/test splitting (preserving class proportions in both sets) and XGBoost's scale_pos_weight parameter set to the negative/positive class ratio, up-weighting the minority Hazardous class during training.

3.3 Model Training

Five supervised classifiers were trained:

  • Logistic Regression (max_iter=1000)
  • Decision Tree (max_depth=10)
  • Random Forest (200 estimators, max_depth=10)
  • Gradient Boosting (200 estimators, lr=0.1, max_depth=5)
  • XGBoost (200 estimators, subsample=0.8, colsample_bytree=0.8)

Evaluation metrics: Accuracy, Weighted F1 Score, Recall for the Hazardous class, and ROC-AUC. The best model was selected automatically by ROC-AUC.

§ 4 System Architecture

The system follows a complete end-to-end pipeline: raw CSV ingestion → preprocessing → feature engineering → model training → evaluation → artifact serialisation → Gradio deployment.

Four Joblib artifacts are serialised: best_model.pkl, scaler.pkl, feature_names.pkl, and model_meta.pkl. The Gradio interface accepts a NASA Neo Reference ID string and returns: prediction label, actual label, verdict (CORRECT ✓ / INCORRECT ✗), hazard probability percentage, and safe probability percentage.

§ 10 Conclusion

This project successfully demonstrates that machine learning can serve as a powerful and reliable tool for asteroid hazard prediction. Among the five models evaluated, XGBoost achieved exceptional performance — ROC-AUC of 0.9992, Accuracy of 99.57%, and Hazardous-class Recall of 99.12% — with stability confirmed by 5-fold cross-validation.

Critically, the high recall for the Hazardous class ensures very few dangerous objects are misclassified as safe — a paramount requirement in safety-critical planetary defence systems where false negatives carry the highest cost.

Feature importance analysis confirmed that Estimated Diameter and Absolute Magnitude are the dominant predictors, aligning with established PHA definition criteria and validating the model's scientific interpretability alongside its statistical accuracy.

The Gradio deployment bridges the gap between theoretical ML and real-world application, enabling researchers and analysts to query hazard assessments without programming expertise. Future work may integrate real-time CNEOS orbital update feeds and extend the feature set with newly available photometric data from next-generation survey instruments.
